# Команды агента Resource Manager

Список доступных команд для автоматизации разработки.

## Основные команды

| Команда | Описание |
|---------|----------|
| `/build` | Сборка backend и frontend |
| `/test` | Запуск тестов и проверка API |
| `/deploy` | Деплой в Docker контейнеры |
| `/check` | Диагностика состояния системы |
| `/fix` | Исправление типичных ошибок |
| `/commit` | Умный коммит с автоопределением типа |
| `/push` | Отправка изменений в репозиторий |
| `/task` | Выполнение задачи из очереди |
| `/phase` | Выполнение фазы разработки |

## Примеры использования

### Начать новую фазу разработки
```
/phase 1
```
Агент выполнит все шаги Фазы 1: создаст структуру backend, entities, миграции, JWT авторизацию.

### Проверить состояние после изменений
```
/check
/test api
```

### Собрать и задеплоить
```
/build
/deploy
```

### Исправить ошибки
```
/fix backend
/fix frontend
```

### Работа с задачами
1. Создай файл `Tasks/add-filter.md` с описанием задачи
2. Запусти `/task`
3. Агент выполнит задачу и отметит её как done

## Расширение команд

Для добавления новой команды создай файл `.claude/commands/your-command.md` со структурой:

```markdown
# /your-command — Описание

## Usage
/your-command [args]

## Workflow
Описание шагов

## Output
Формат вывода
```

## Автономная работа

Для полностью автономной работы агента:

1. Настрой `settings.local.json` с полными правами
2. Создай задачи в папке `Tasks/`
3. Запусти агента с командой `/task`
4. Агент будет последовательно выполнять задачи

## Порядок фаз

1. **Phase 1** — Backend Core (entities, auth, employees CRUD)
2. **Phase 2** — Backend Admin (columns, dictionaries, users, history)
3. **Phase 3** — Frontend Core (login, table, form)
4. **Phase 4** — Frontend Admin + Docker (admin UI, filters, deploy)

Проверяй прогресс командой `/phase status`.
