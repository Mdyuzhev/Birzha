# Resource Manager — Claude Agent Configuration

## Описание

Конфигурация для Claude Code агента, позволяющая автономно разрабатывать систему управления ресурсами (сотрудниками).

## Содержимое

```
resource-manager/
├── .claude/
│   ├── CLAUDE.md           # Полное описание проекта для агента
│   ├── settings.json       # Разрешения для команд
│   ├── settings.local.json # Локальные настройки с полными правами
│   └── commands/           # Команды автоматизации
│       ├── README.md       # Документация по командам
│       ├── build.md        # Сборка проекта
│       ├── test.md         # Запуск тестов
│       ├── deploy.md       # Деплой в Docker
│       ├── check.md        # Диагностика системы
│       ├── fix.md          # Исправление ошибок
│       ├── commit.md       # Умный коммит
│       ├── push.md         # Push изменений
│       ├── task.md         # Выполнение задач из очереди
│       └── phase.md        # Выполнение фаз разработки
├── docker-compose.yml      # Конфигурация Docker
├── init-data.sql           # Начальные данные
└── README.md               # Этот файл
```

## Быстрый старт

### 1. Подготовка
```bash
# Распаковать архив
unzip resource-manager-claude.zip
cd resource-manager

# Инициализировать git (если нужно)
git init
```

### 2. Запуск Claude Code
```bash
# Запустить агент в директории проекта
claude
```

Агент автоматически прочитает `.claude/CLAUDE.md` и будет знать контекст задачи.

### 3. Выполнение фаз разработки

Проект разбит на 4 фазы. Запускай их последовательно:

```
/phase 1    # Backend: entities, auth, employees CRUD
/phase 2    # Backend: admin API, history, roles
/phase 3    # Frontend: login, table, form
/phase 4    # Frontend: admin UI, Docker
```

После каждой фазы проверяй результат:
```
/check
/test api
```

## Автономная работа

Для полностью автономного выполнения используй очередь задач:

1. Создай папку `Tasks/` в корне проекта
2. Добавь файлы задач в формате `.md`:

```markdown
# Добавить фильтрацию

## Описание
Реализовать фильтрацию сотрудников по статусу

## Критерии готовности
- [ ] Backend endpoint принимает параметр status
- [ ] Frontend отображает dropdown
```

3. Запусти агент с командой `/task`

Агент будет последовательно выполнять задачи, отмечая завершённые.

## Доступные команды

| Команда | Описание |
|---------|----------|
| `/build` | Сборка backend и frontend |
| `/test` | Запуск тестов |
| `/deploy` | Деплой в Docker |
| `/check` | Диагностика системы |
| `/fix` | Исправление типичных ошибок |
| `/commit` | Умный коммит |
| `/push` | Push изменений |
| `/task` | Выполнение задачи из очереди |
| `/phase` | Выполнение фазы разработки |
| `/phase status` | Прогресс по фазам |

## Настройки

### settings.json
Определяет разрешённые команды для безопасной работы. По умолчанию разрешены: mvn, npm, docker, git, curl и базовые unix-утилиты.

### settings.local.json
Локальные настройки с расширенными правами для разработки. Включает полный доступ к Bash и файловой системе.

## Расширение

Для добавления новых команд создай файл `.claude/commands/your-command.md` и следуй формату существующих команд.
